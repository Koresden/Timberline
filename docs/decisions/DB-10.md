## 3. The privacy story is clean (verified)

Apple's privacy-manifest enforcement (since **May 1, 2024**) is **conditional**: an app must
declare *required-reason APIs* only if it (or a bundled SDK) actually uses them.

- **Timberline uses no required-reason-API plugin.** Its only Capacitor packages are
  `@capacitor/core` + `cli` + `ios`. State is `zustand` + `localStorage` (web storage — not a
  required-reason API). **No `@capacitor/preferences`** (would need UserDefaults `CA92.1`),
  **no `@capacitor/filesystem`** (would need FileTimestamp `C617.1`).
- **Capacitor *core* already ships its own privacy manifest** (verified against current source:
  `tracking=false`, empty arrays) — that satisfies the SDK-level obligation. We don't write one
  for Capacitor.
- **Net:** an app-level `PrivacyInfo.xcprivacy` is **not strictly required**. Adding a
  defensive, effectively-empty one (`NSPrivacyTracking=false`, empty
  collected-data / tracking-domains / accessed-API arrays) is optional belt-and-suspenders and
  cheap — recommended for explicitness, not because review demands it.
- **App Privacy "nutrition" label:** legitimately **"Data Not Collected"** — fully offline,
  no accounts, no analytics; on-device-only processing is not "collection" per Apple.

> Verifier note: both fact-checkers confirmed the manifest mechanics and the single-1024 icon
> rule against primary sources; the only correction was that the manifest is *conditional*, not
> universal — which is exactly why Timberline's no-plugin build gets off easy.

## 6. Safety gate — the store listing is a posture surface

DB-8 bucket **C** extends to marketing: **the listing must never imply detection,
authorization, or lockout.** The safety-auditor reviews the description, subtitle, keywords,
screenshot plan, category, and age rating before any of it ships. Constraints (from the
posture review):

- **Forbidden in copy/keywords/screenshots:** "detects people / live area scan / person in
  danger zone / N breach", "tells you when it's safe to cut / go-ahead", "felling locked until
  clear / unlocks felling", "begin cut sequence", "guarantees a safe fall", "OSHA-certified",
  "replaces an arborist", "AR / camera detection", "instrument-grade / exact height". **Never
  screenshot `design/screens/04-danger-zone.png`** — that scanner was intentionally never built
  (DB-8 §C); depicting it would advertise a product that must not exist.
- **Must say, up front (not fine print):** planning aid, **not** authorization; assumes a
  trained operator with PPE; recommends a **professional arborist** and drops to
  information-only for big / leaning / dead / near-structure trees; height is an **estimate
  with an error band**; the operator stays responsible.
- **Category:** Utilities (primary), Reference (optional secondary) — **never Medical** or any
  safety/compliance/authorization-device framing.
- **Age rating:** posture-driven **17+ / highest adult band** — an adult, trained-operator
  tool, never kid-suitable (it concerns chainsaw felling).
- **Honest one-liner (draft):** *"Timberline is an offline planning aid for tree-felling — it
  helps a trained operator with PPE think through height, cut, and fall zone, and recommends a
  professional arborist for big, leaning, dead, or near-structure trees. It does not detect
  people, decide when it's safe, or authorize the cut."*
- Screenshots may show: the Measure sighting **illustration** (labeled, not AR), the Height
  stat with its **± ΔH worst-case** band, the Plan cut card, the Simulate **danger** radius
  read **as danger**, the persistent **SafetyBanner**, and the **refer-professional** takeover.

runtime bundle), so it does not expand the app's runtime surface. It is the tool the official

Xcode — correct but tedious/error-prone for the splash dark variant. Requesting the nod to add
it as a dev dependency.
